# 智能对话助手历史提问导航设计

## 目标

在智能对话助手的当前会话中增加历史提问快速导航。用户打开浮动导航后，可以浏览自己此前发送的问题；点击问题后，聊天区平滑滚动到原消息并提供短暂高亮反馈。

## 范围

- 仅展示当前会话中 `role === 'user'` 的消息。
- 不展示助手回答摘要。
- 不跨会话搜索，不新增后端接口。
- 不改变消息持久化结构。
- 同时支持已持久化消息和本轮尚未取得数据库 ID 的乐观消息。

## 交互结构

采用右侧浮动导航方案：

```text
聊天抽屉
  ├─ 消息滚动区
  │    ├─ 用户问题 A  ← 可定位目标
  │    ├─ 助手回答 A
  │    ├─ 用户问题 B  ← 可定位目标
  │    └─ 助手回答 B
  │
  └─ 右侧 44×44 浮动按钮
          └─ 提问列表弹层
                ├─ 用户问题 A
                └─ 用户问题 B
```

### 打开与关闭

- 点击浮动按钮切换弹层。
- 点击弹层外关闭。
- 按 `Escape` 关闭。
- 切换会话或关闭聊天抽屉时关闭。
- 会话没有用户提问时按钮隐藏，避免空入口。

### 定位反馈

1. 点击提问条目。
2. 根据稳定消息键找到对应 `.msg-row`。
3. 调用 `scrollIntoView({ behavior: 'smooth', block: 'center' })`。
4. 为目标消息增加高亮类。
5. 约 1.5 秒后移除高亮类。
6. 用户开启 `prefers-reduced-motion` 时使用即时滚动并取消明显位移动画。

## 组件设计

### `ChatQuestionNavigator.vue`

职责：

- 接收已经过滤和格式化的用户提问列表。
- 渲染浮动按钮与弹层。
- 管理展开/收起、外部点击和 Escape。
- 发出 `locate(questionKey)` 事件，不直接操作消息 DOM。

接口：

```text
props:
  questions: Array<{ key, content, createdAt }>
  activeKey: String

emits:
  locate(key)
```

### `ChatMessageList.vue`

职责：

- 为每条消息生成稳定定位键。
- 把定位键写入消息行 `data-message-key`。
- 通过 `defineExpose` 暴露 `locateMessage(key)`。
- 执行滚动与临时高亮。

稳定键优先使用数据库消息 ID；没有 ID 的乐观消息使用 `role + createdAt + index`。同一轮渲染中导航项和消息行必须调用同一键生成规则。

### `ChatAssistantDrawer.vue`

职责：

- 从 `chatStore.messages` 计算当前用户提问列表。
- 挂载导航组件。
- 调用 `ChatMessageList.locateMessage()`。
- 在会话切换、抽屉关闭时重置导航状态。

## 视觉规范

- 触发按钮交互区域不小于 44×44px。
- 使用现有聊天抽屉的蓝灰色视觉令牌和 SVG 线性图标。
- 弹层宽度桌面端约 260–300px，最大高度不超过聊天区的 60%，内部滚动。
- 条目单行省略，完整内容通过 `title` 和可访问名称提供。
- hover、focus-visible 和 active 状态均可区分；不能只依赖颜色表示选中状态。
- 移动端弹层左右边界限制在聊天抽屉内，不产生页面级横向滚动。

## 状态与异常

- 消息为空或只有助手欢迎语：不显示导航。
- 目标消息因会话切换而不存在：静默关闭弹层，不抛出界面错误。
- 消息流式更新：导航仅依赖用户消息，不因助手 chunk 重复重算定位键。
- 新问题发送后：列表自动新增该问题，现有聊天自动滚底逻辑保持不变。
- 快速连续点击不同问题：清除前一个高亮计时器，只保留最后目标的高亮。

## 无障碍

- 浮动按钮使用原生 `button` 和 `aria-label="查看历史提问"`。
- `aria-expanded` 反映弹层状态，`aria-controls` 指向列表。
- 每个问题使用原生按钮，可通过 Tab 和 Enter/Space 操作。
- Escape 关闭后焦点返回触发按钮。
- 保留明显的 `focus-visible` 焦点环。

## 测试

新增前端契约测试，至少验证：

1. 只从消息中筛选用户提问。
2. 导航按钮满足可访问属性要求。
3. 消息行存在稳定的定位标识。
4. 点击导航项触发定位方法。
5. 定位调用 `scrollIntoView` 并设置临时高亮。
6. Escape、外部点击和会话切换能关闭弹层。
7. 样式包含移动端边界和 reduced-motion 处理。

## 验收标准

- 当前会话存在用户消息时显示浮动提问导航。
- 列表内容与用户消息数量、顺序一致。
- 点击任一条目可以准确滚动到原问题。
- 原问题有短暂且清晰的高亮反馈。
- 不影响消息发送、SSE 流式输出、自动滚底和会话切换。
- 键盘操作、移动端和 reduced-motion 均可用。

# 智能对话助手历史提问导航实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在当前聊天会话内提供用户历史提问浮动列表，并支持点击问题后准确滚动和高亮原消息。

**Architecture:** 新建纯展示与交互组件 `ChatQuestionNavigator.vue`；`ChatAssistantDrawer` 从 Store 消息计算问题列表并协调定位；`ChatMessageList` 负责稳定消息键、DOM 定位和高亮。功能完全在前端完成，不修改后端协议。

**Tech Stack:** Vue 3 Composition API、Pinia、CSS、Node.js Contract Tests

---

### Task 2：实现消息定位能力

**Files:**
- Modify: `frontend/src/components/chat/ChatMessageList.vue`
- Test: `frontend/tests/chat-question-navigator-contract.test.cjs`

- [ ] **Step 1：为消息行增加稳定键**

增加统一函数：

```javascript
const messageKey = (item, idx) => item.id
  ? `message-${item.id}`
  : `message-${item.role}-${item.createdAt || 'pending'}-${idx}`
```

模板消息行增加：

```vue
:data-message-key="messageKey(item, idx)"
```

- [ ] **Step 2：暴露定位方法**

```javascript
const locateMessage = (key) => {
  const target = listRef.value?.querySelector(`[data-message-key="${CSS.escape(key)}"]`)
  if (!target) return false
  target.scrollIntoView({ behavior: reducedMotion() ? 'auto' : 'smooth', block: 'center' })
  target.classList.add('question-locate-highlight')
  // 清除旧计时器并在 1500ms 后移除 class
  return true
}
defineExpose({ locateMessage, messageKey })
```

- [ ] **Step 3：增加高亮和 reduced-motion 样式**

高亮使用 outline/background，不改变消息尺寸；`prefers-reduced-motion: reduce` 下关闭过渡和位移动画。

const questionItems = computed(() => chatStore.messages
  .map((message, index) => ({ message, index }))
  .filter(({ message }) => message.role === 'user')

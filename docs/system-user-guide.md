# 集装箱门把手工业缺陷检测系统使用说明

本文档面向系统操作人员、质检人员、模型维护人员和管理员，说明如何使用本系统完成图像采集、缺陷检测、质检复核、结果追溯、模型管理和智能助手辅助操作。

## 1. 系统定位

本系统用于集装箱门把手工业缺陷检测，核心目标是把“图像采集、模型检测、质检复核、缺陷证据、工单追溯、批次追溯、模型管理、设备人员管理”串成一个完整闭环。

典型业务流程如下：

1. 采集或上传门把手图像。
2. 创建检测任务并调用模型推理。
3. 查看检测结果、缺陷框、置信度和严重等级。
4. 对可疑或异常结果进行人工质检复核。
5. 将复核结果沉淀到质检队列、缺陷证据库、工单追溯和批次追溯。
6. 根据数据反馈优化模型和产线管理。

## 2. 访问与登录

### 2.1 启动后端

在项目根目录执行：

```powershell
.\mvnw.cmd spring-boot:run
```

默认后端地址：

```text
http://localhost:8080
```

### 2.2 启动前端

进入前端目录后启动：

```powershell
cd frontend
npm.cmd run dev
```

默认前端访问地址以终端输出为准，通常是：

```text
http://localhost:5173
```

### 2.3 登录系统

打开前端地址后进入登录页，输入账号密码进入系统。登录成功后进入首页总览。

如果未登录直接访问业务页面，系统会自动跳转到登录页。

## 3. 首页总览

入口：

```text
首页总览
```

首页用于查看平台运行态势，包括检测任务、设备状态、模型状态、质检情况等综合信息。建议管理员和班组负责人优先从首页观察当天整体运行情况。

## 4. 检测业务

检测业务包含：

```text
图像采集上传
检测工作台
检测记录
```

### 4.1 图像采集上传

入口：

```text
检测业务 -> 图像采集上传
```

用于上传原始检测图片并创建检测任务。

建议填写或确认以下信息：

- 图片文件：门把手原图。
- 批次信息：用于后续批次追溯。
- 工单信息：用于后续工单追溯。
- 设备信息：用于定位采集来源。
- 操作人员：用于明确责任链路。

上传完成后，系统会将图片和任务信息纳入检测链路。

### 4.2 检测工作台

入口：

```text
检测业务 -> 检测工作台
```

检测工作台用于执行模型检测和查看当前任务状态。

常用操作：

- 选择模型：可选择指定模型，不选择时使用默认模型。
- 设置置信度阈值：控制检测结果显示和判断灵敏度。
- 单图检测：适合快速验证单张图片。
- 查看任务列表：观察待检测、检测中、已完成、失败等状态。
- 刷新任务：同步最新任务进度。

结果查看重点：

- 检测类别。
- 缺陷位置。
- 置信度。
- 严重等级。
- 检测图片证据。
- 是否需要进入质检复核。

### 4.3 检测记录

入口：

```text
检测业务 -> 检测记录
```

检测记录用于查询历史检测结果和任务报告。

常见用途：

- 按任务、图片、批次、工单查询历史检测结果。
- 查看检测是否成功。
- 复查模型输出的缺陷结果。
- 导出或查看质检报告。
- 对历史异常任务进行追踪。

## 5. 质检追溯

质检追溯包含：

```text
质检队列
缺陷证据库
工单追溯
批次追溯
```

这些页面复用统一的“图像检测中心”业务页面，但在左侧导航中分为独立的质检追溯模块，便于质检人员按工作职责访问。

### 5.1 质检队列

入口：

```text
质检追溯 -> 质检队列
```

质检队列用于处理需要人工复核的检测任务。

常见状态：

- 待分配：任务尚未指定质检人员。
- 待复核：等待质检人员确认结果。
- 复核中：正在处理。
- 已通过：检测结果被确认。
- 已驳回或需返工：检测结果需要修正或重新处理。

不同状态下可执行的操作不同，例如分配、复核、处置、返工提交、导出报告等。操作人员应根据当前状态选择可用动作。

### 5.2 缺陷证据库

入口：

```text
质检追溯 -> 缺陷证据库
```

缺陷证据库用于集中查看缺陷图片和检测证据。

建议关注：

- 缺陷类别。
- 严重等级。
- 置信度。
- 缺陷位置。
- 所属批次。
- 所属工单。
- 使用模型。
- 采集设备。

该模块适合用于质量分析、客户追责、模型复盘和缺陷样本积累。

### 5.3 工单追溯

入口：

```text
质检追溯 -> 工单追溯
```

工单追溯用于按工单号查看完整检测链路。

可追溯内容包括：

- 工单关联的检测任务。
- 图片采集记录。
- 模型检测结果。
- 质检复核记录。
- 操作人员。
- 设备来源。
- 缺陷证据。
- 报告导出。

当需要定位某个生产任务的质量问题时，优先使用工单追溯。

### 5.4 批次追溯

入口：

```text
质检追溯 -> 批次追溯
```

批次追溯用于按批次号查看一批产品的整体检测质量。

常见用途：

- 查看批次内检测总量。
- 统计缺陷数量和缺陷类型分布。
- 定位异常设备或异常模型。
- 对批次质量进行复核。
- 支撑出厂质量报告。

## 6. 模型中心

模型中心包含：

```text
模型管理
上传模型
```

### 6.1 模型管理

入口：

```text
模型中心 -> 模型管理
```

用于查看和维护检测模型。

常见操作：

- 查看模型列表。
- 查看模型版本。
- 校验模型文件。
- 发布模型。
- 设置默认模型。
- 查看模型使用统计。
- 查看模型操作日志。

建议只将经过验证的模型发布到生产检测链路，避免未验证模型影响检测准确率。

### 6.2 上传模型

入口：

```text
模型中心 -> 上传模型
```

用于导入新的模型版本。

上传时建议确认：

- 模型名称。
- 版本号。
- 模型文件格式。
- 适用缺陷类别。
- 兼容的推理服务。
- 模型说明。

上传后建议先校验，再发布，再设置为默认模型。

## 7. 产线资源

产线资源包含：

```text
设备管理
设备使用记录
人员管理
```

### 7.1 设备管理

入口：

```text
产线资源 -> 设备管理
```

用于维护采集设备、检测设备和产线设备信息。

建议记录：

- 设备编号。
- 设备名称。
- 设备位置。
- 在线状态。
- 负责人。
- 维护状态。

设备信息会影响检测任务追溯，建议保持准确。

### 7.2 设备使用记录

入口：

```text
产线资源 -> 设备使用记录
```

用于查看设备调用、使用和维护记录。

适合排查：

- 某台设备是否频繁产生异常结果。
- 某段时间设备是否离线。
- 检测任务是否来自指定采集端。

### 7.3 人员管理

入口：

```text
产线资源 -> 人员管理
```

用于维护采集人员、质检人员、维修人员和管理员信息。

人员信息会参与操作日志、质检责任和追溯链路。

## 8. 智能对话助手

入口：

```text
页面右下角智能助手入口
```

智能助手用于辅助查询系统数据、解释业务含义、定位功能入口和生成操作建议。

可询问示例：

```text
今天有哪些待复核任务？
帮我查看某个批次的检测结果。
模型管理在哪里？
质检队列有哪些状态？
某个工单的追溯链路怎么看？
```

安全说明：

- 智能助手已加入敏感信息脱敏。
- 智能助手的文件扫描范围受白名单限制。
- 智能助手接口有请求限流。
- 外部模型调用已配置超时。

注意：智能助手用于辅助操作和解释，不应替代质检人员的最终判断。

## 9. 推荐操作流程

### 9.1 日常检测流程

1. 进入“图像采集上传”上传图片。
2. 进入“检测工作台”查看任务执行状态。
3. 检测完成后进入“检测记录”查看结果。
4. 对需要人工确认的任务进入“质检队列”复核。
5. 复核后在“缺陷证据库”沉淀缺陷样本。
6. 按需通过“工单追溯”或“批次追溯”生成追溯结论。

### 9.2 模型更新流程

1. 进入“上传模型”导入新模型。
2. 在“模型管理”执行模型校验。
3. 确认模型信息和版本。
4. 发布模型。
5. 设置为默认模型。
6. 观察新模型的检测效果和质检反馈。

### 9.3 异常排查流程

1. 先查看“首页总览”确认整体状态。
2. 进入“检测记录”确认异常任务。
3. 进入“质检队列”查看是否需要人工处理。
4. 进入“工单追溯”或“批次追溯”定位来源。
5. 如果异常集中在某台设备，查看“设备管理”和“设备使用记录”。
6. 如果异常集中在某个模型，查看“模型管理”和模型使用统计。

## 10. 常见问题

### 10.1 登录后看不到数据

可能原因：

- 当前数据库没有初始化演示数据。
- 当前筛选条件过窄。
- 后端服务未正常连接数据库。
- 当前账号权限不足。

建议先清空筛选条件，再刷新页面。

### 10.2 检测任务一直未完成

可能原因：

- 推理服务未启动。
- 远程检测服务不可达。
- OSS 或图片路径配置异常。
- 模型文件不可用。

建议查看后端日志，并确认推理服务和模型状态。

### 10.3 质检队列没有数据

可能原因：

- 当前没有需要复核的检测任务。
- 队列状态筛选不匹配。
- 检测任务未进入质检流程。

建议切换到“全部可处理”或刷新队列。

### 10.4 工单追溯或批次追溯查不到结果

可能原因：

- 工单号或批次号输入错误。
- 上传图片时未绑定工单或批次。
- 当前任务尚未完成检测。

建议从检测记录中复制准确的工单号或批次号再查询。

### 10.5 模型重新校验失败

可能原因：

- 模型文件缺失。
- 文件格式不符合要求。
- 模型元数据不完整。
- 推理运行环境不兼容。

建议检查模型详情、后端日志和模型文件路径。

| --- | --- |
| 管理员 | 首页总览、模型中心、设备管理、人员管理 |

## 12. 使用注意事项

- 上传图片前尽量保证图像清晰、角度稳定、目标完整。
- 关键生产任务建议保留工单号和批次号。
- 模型发布前应完成校验，避免影响生产检测。
- 质检复核结果应及时处理，避免队列积压。
- 追溯查询建议优先使用准确的工单号或批次号。
- 智能助手可以辅助查询，但最终质量结论应以质检流程记录为准。

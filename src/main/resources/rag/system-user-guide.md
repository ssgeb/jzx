# 集装箱门把手工业缺陷检测系统使用手册知识库

## 系统定位

系统用于集装箱门把手工业缺陷检测，覆盖图像采集、模型检测、检测记录、质检复核、缺陷证据、工单追溯、批次追溯、模型管理、设备管理、人员管理和智能助手。

推荐业务闭环是：图像采集上传 -> 检测工作台执行模型检测 -> 检测记录查看历史结果 -> 质检队列人工复核 -> 缺陷证据库沉淀证据 -> 工单追溯和批次追溯定位质量链路。

## 检测业务

检测业务包含图像采集上传、检测工作台、检测记录。

图像采集上传用于上传门把手原图，创建检测任务，并补充批次号、工单号、采集设备、采集人员等追溯字段。

检测工作台用于选择模型、设置置信度阈值、执行单图或任务检测，查看待检测、检测中、已完成、失败等任务状态。

检测记录用于查询历史检测任务，查看检测类别、缺陷位置、置信度、严重等级、检测图片证据和报告。

## 质检追溯

质检追溯包含质检队列、缺陷证据库、工单追溯、批次追溯。

质检队列用于处理待分配、待复核、复核中、已通过、已驳回、需返工、需复检等状态任务。不同状态下可执行分配、复核、处置、返工提交、导出报告等不同操作。

缺陷证据库用于集中查看缺陷图片、缺陷类别、缺陷位置、缺陷面积、置信度、严重等级、所属批次、所属工单、使用模型和采集设备。

工单追溯用于按工单号查看检测任务、图片采集记录、模型检测结果、质检复核记录、操作人员、设备来源、缺陷证据和报告。

批次追溯用于按批次号查看检测总量、缺陷数量、缺陷类型分布、异常设备、异常模型和批次质量结论。

## 模型中心

模型中心包含模型管理和上传模型。

模型管理用于查看模型列表、版本、校验状态、发布状态、默认模型、使用统计和操作日志。生产检测建议只使用已校验并发布的模型。

上传模型用于导入新模型版本。上传后应先执行校验，再发布模型，最后根据需要设置为默认模型。

## 产线资源

产线资源包含设备管理、设备使用记录、人员管理。

设备管理用于维护采集设备、检测设备、设备编号、设备位置、在线状态、负责人和维护状态。

设备使用记录用于查看设备调用、采集、检测和维护记录，排查某台设备是否频繁产生异常结果。

人员管理用于维护采集人员、质检人员、维修人员和管理员信息，支撑操作日志和质量责任追溯。

## 智能助手

智能助手可以帮助用户查询系统功能入口、解释模块用途、查看待复核任务、查询工单或批次追溯方式、说明模型管理流程和给出下一步操作建议。

可询问示例：今天有哪些待复核任务；帮我查看某个批次的检测结果；模型管理在哪里；质检队列有哪些状态；某个工单的追溯链路怎么看。

## 常见问题

登录后看不到数据时，先清空筛选条件并刷新页面，再检查后端服务、数据库连接和账号权限。

检测任务一直未完成时，应检查推理服务、远程检测服务、OSS 图片路径和模型状态。

质检队列没有数据时，可能当前没有需要复核的任务，或队列状态筛选不匹配，建议切换到全部可处理后刷新。

工单追溯或批次追溯查不到结果时，应确认工单号或批次号准确，且上传图片时已绑定追溯字段。

模型重新校验失败时，应检查模型文件是否存在、格式是否正确、元数据是否完整、推理环境是否兼容。

# 智能助手 RAG 能力说明

## RAG 检索流程

智能助手支持使用 RAG 检索系统知识库。用户提问后，系统会先对问题进行提示词重写，生成更适合知识库检索的查询语句，然后使用 ChromaDB 进行向量检索，最后对候选知识片段进行 rerank 重排，将最相关内容注入智能助手 Agent 上下文。

完整链路是：用户原始问题 -> LLM 查询重写 -> 向量检索候选片段 -> LLM rerank -> 注入 Agent 上下文 -> 生成最终回答。

## 查询重写

查询重写用于把口语化、模糊、不完整的问题转换为更适合检索的表达。

例如用户问“这个质检咋处理”，可以重写为“质检队列状态、人工复核、缺陷确认、返工复检、处置流程”。

例如用户问“模型怎么弄”，可以重写为“模型管理、上传模型、模型校验、模型发布、默认模型设置、模型使用统计”。

## Rerank 重排

Rerank 用于从 Chroma 返回的多个候选片段中，选择最能回答用户问题的片段。

重排时应优先选择直接说明功能入口、操作流程、状态含义、故障排查步骤的片段。

如果候选片段只包含相同关键词但无法回答问题，应降低排名。

## 防止错误回答和胡乱编造

智能助手回答必须基于当前系统数据、RAG 知识库片段和用户提供的信息。

如果知识库没有检索到相关片段，或者实时系统数据不足，助手应该明确说明“我没有在当前系统数据或知识库中找到足够依据”，并告诉用户需要补充任务编号、工单号、批次号、页面名称或筛选条件。

助手不得编造不存在的页面入口、按钮名称、模型版本、任务编号、工单号、批次号、统计数值、设备状态、人员信息或检测结论。

如果只能做推测，必须使用“可能”表述，并给出验证路径，例如建议用户进入检测记录、质检队列、模型管理、设备使用记录、工单追溯或批次追溯确认。

## Chroma 向量文档

助手文档、用户使用手册、Chroma RAG 配置说明和 README 都可以作为向量文档写入 Chroma。

系统启动时会读取配置中的 rag-sources，将 Markdown 按标题和长度切分成知识片段，生成 embedding 后 upsert 到 Chroma collection。

如果 Chroma 不可用，系统自动回退本地关键词检索，智能助手仍可工作。

## 可回答的问题

智能助手可以通过 RAG 回答：系统有哪些模块、检测工作台怎么用、质检队列有哪些状态、工单追溯怎么看、批次追溯怎么看、模型如何上传和发布、设备管理做什么、智能助手 RAG 如何配置、Chroma 怎么启动、RAG 为什么会回退。
